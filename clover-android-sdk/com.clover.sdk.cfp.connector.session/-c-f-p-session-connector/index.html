Used to send data from the customer device/screen to a POS device/service. If the message can be sent remotely, it will be send to the POS/MFD and local listeners will NOT be notified. If there isn't a remote message conduit, then listeners on the current device will be notified. This allows the same call and listener to be used in both a tethered, and non-tethered, scenario.
