ReceiptContract

public final class ReceiptContract

A class defining the contract to be implemented by content providers that wish to provide additional content on customer receipts.

To register a receipt registration content provider with the Clover platform, use the register method.

Receipt registration content providers must support either, or both the CONTENT_TYPE and CONTENT_TYPE content types.

If the provider supports the CONTENT_TYPE content type, it must implement the query method. If the returned cursor contains the column TEXT, the string value of the column is added to the receipt. The content provider may return a null cursor, or a cursor that does not contain the TEXT column to indicate that the receipt should not be modified.

If the content provider supports the CONTENT_TYPE content type, it must implement the openFile method, and the returned file descriptor value must either:

  • Point to a file that contains bitmap data that can be decoded using decodeStream. For example, it can be a PNG, GIF, or JPG encoded file.
  • Be null, indicating that the content provider declined to add image content to this receipt.
When called, a receipt registration content provider is passed the PARAM_* query parameters listed below. This may be used as input to other Clover services to obtain additional information about the order, customer, etc. Receipt registration content providers have a maximum of 2 seconds to return receipt content. It is advisable to stay well below this limit. You must not perform long-running tasks such as network I/O. If the receipt registration content provider repeatedly fails to respond in the allotted time, throws exceptions, or otherwise fails to return valid content as specified above, the provider may be temporarily or permanently disabled.

It is recommended that apps registering receipt registration providers implement a pre-uninstall hook and unregister the content provider. See: ACTION_APP_PRE_UNINSTALL for more information.
